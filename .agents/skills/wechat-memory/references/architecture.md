# 分层与存储

```text
用户提供的结构化 JSON
→ 原始记录与版本
→ 人物 / 会话 / 群成员 / 消息 / 身份分类
├→ QMD 派生检索文档 / vector index
├→ 人物画像（可选、可删除、可重建）
→ SQL / FTS / vector 混合召回
→ 当前 Agent 查询
→ 人脉互动图
```

## 两个数据库

- `crm.sqlite`：事实层。保存原始记录版本、人物、身份角色、会话、群成员、消息和全文索引。
- `analysis.sqlite`：派生层。保存人物摘要、画像事实、证据快照与分析运行。
- `search-docs/` + QMD named index：派生检索层。把高信息文本切成小型对话窗口，并为长消息生成
  单条重点卡片；只保存安全可读内容和消息 ID，可随时重建，collection/index 独立。

默认位于 `~/Library/Application Support/wechat-memory/`，不进入 Obsidian，也不进入 Git。

画像是高层 materialized view，不是查询门槛。原消息负责完整证据；画像负责压缩、稳定事实、分类与后续跨人物关系。

## 身份不是好友表

一个身份可以同时是：私聊对象、群内见过、公众号或服务号、本人、未知。`known_identities` 是导入数据中出现的所有身份，不能等同好友数，也不能把各分类相加。

## 可视化语义

- 人物节点：本地消息中出现的人。
- 群节点：所选时间窗内活跃群聊。
- 你—人物：存在私聊消息。
- 群—人物：该人物在该群中存在本地可见发言。

线宽只按可见消息量绘制，不是关系评分，不代表双方真实亲密程度。
